Important concepts

Lesson 1: The Basics of the Web and HTML

World Wide Web

The World Wide Web is a huge collection of HTML documents. It was invented in the 1990s and now it contains over 30 billion web pages.

This is how it works. When you visit a web page your computer sends a HTTP request to a server via the Internet. The server finds the HTML document with all the files for the page and sends it back to your computer. The browser on your computer interprets the information sent by the server and shows you the web page. Here is a good video about the components of the web.

HTML

HTML (Hypertext Markup Language) is a special code for computers. HTML documents contain text content (what you actually see on the page) and markup (how the content is arranged).

Tags

HTML markup is made of different tags. Together with content they make up different types of elements. Here is a video about structure of HTML documents.

Tags are essential and you should be very carefull when writing HTML because computers are "stupid". Tags are instructions for them so even a smallest mistake can lead to a big problem.

Inline and block elements

HTML elements can be inline or block. Block elements unlike inline ones create an "invisible box" around the content. The concept of boxes is very important in web programming.

Lesson 2: Creating a Structured Document with HTML

Developer Tools

Developer tools in Chrome browser shows the structure of the page. In Developer tools it is clearly seen that HTML elements form boxes on the page.

Page Structure

The structure of any page is "tree-like". It means HTML elements can be inside other elements.

If you look at the structure of the page in Developer tools or any other tools you can see sideway triangles next to some of the elements. They indicate the "branching" structure and that these elements contain other elements. The more deeply "nested" in the structure an element is, the more indented it is in the code.

"Boxifying" design

"Everything on the Web is boxes". Before writing a code of a web page it is important to "boxify" the design first - to cut your future page into boxes.

Lesson 3: Adding CSS Style to HTML Structure

CSS

CSS (Cascading Style Sheets) is a language which allows programmers apply style to HTML elements. It also helps them avoid repetition.

Avoiding repetition

"Don't repeat yourself" (DRY) is an important principle for software developers. The key feature of CSS that helps not to write the same code over and over is inheritance. It means that the style or rules applied to a parent element is inherited by all its child elements.
